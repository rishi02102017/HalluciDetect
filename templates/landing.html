FAQ

Frequently Asked Questions

Common questions about hallucination detection and the platform

What is an LLM hallucination?

A hallucination occurs when an AI model generates information that sounds plausible but is factually incorrect, fabricated, or inconsistent with the provided context. This can include made-up facts, incorrect dates, non-existent references, or logically inconsistent statements.

How is the hallucination score calculated?

The score combines three detection methods: fact verification against knowledge bases (Wikipedia API), semantic similarity comparison with reference answers using TF-IDF, and rule-based pattern detection for common hallucination indicators. Each method contributes to a weighted final score.

Which LLM models are supported?

HalluciDetect supports models from OpenAI (GPT-4o, GPT-4, GPT-3.5), Anthropic (Claude 3.5, Claude 3), and open-source models via OpenRouter including Llama 3.1, Gemma 2, and Mistral 7B. You can compare hallucination rates across different providers.

Is my data secure?

Your evaluation data is stored securely and associated only with your account. We do not share or sell your data. The platform is open source, so you can audit the code or self-host if you prefer complete control over your data.

Can I use this for production systems?

Yes. HalluciDetect is designed for both research and production use cases. You can integrate it into your CI/CD pipeline for automated testing, use the batch import feature for large-scale evaluations, or monitor hallucination trends over time.

Is there an API available?

Yes, all evaluation functionality is accessible via REST API endpoints. You can programmatically submit evaluations, retrieve results, and export data. Authentication is required via your account credentials.
